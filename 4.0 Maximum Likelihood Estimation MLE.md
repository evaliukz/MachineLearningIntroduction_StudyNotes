# Maximum Likelihood Estimation（MLE）

### 1. MLE到底是什么？

MLE：Maximum Likelihood Estimation

中文：最大似然估计

核心思想：假设数据已经发生了，哪组参数最可能产生这些数据？

举个最简单的例子：你捡到一枚硬币。

连续扔10次：正 正 正 正 正 正 正 反 正 正

统计：9个正面 1个反面

现在问：这枚硬币正面概率 p 是多少？

### 2. MLE在干嘛

MLE会尝试：p=0.1 ? p=0.2 ? p=0.5 ? p=0.9 ?

看哪个最可能产生：9个正面 1个反面 如果：p=0.9 出现这种结果非常合理。

如果：p=0.1 几乎不可能。 所以：MLE：p^=0.9

### 3. 什么是 Likelihood？
  
Likelihood：P(D∣θ)  表示：给定参数 θ （就是w和b的组合），产生数据 D 的概率

例如：参数p=0.9 数据：9 heads  1 tail

Likelihood：P(9H,1T∣p=0.9)

MLE：就是找argθ 的max P(D∣θ)

### 4. 为什么机器学习都在做MLE

因为训练数据已经给你了。你不知道真正参数是多少。所以：你只能反推：什么参数最可能生成这些数据？这就是：Parameter Estimation

### 5. MLE和Gradient Descent的关系？

很多机器学习初学者都会把：MLE， Gradient Descent误以为是两种不同的方法。

其实：*MLE是目标，Gradient Descent是求解方法*

(1). MLE到底是什么？

MLE说：θ^=argθmaxP(D∣θ) 意思：找到最可能产生数据D的参数θ，这是一个优化目标，不是算法。

(2). 怎么找到这个参数？有两种情况。

情况1：有解析解（Closed Form Solution）

例如：估计高斯分布均值。

数据：1,2,3,4,5

MLE推导：直接得到：

μ^ = ∑x_i / n

直接算：m_u = np.mean(x)

结束。不需要：gradient descent, learning rate, iteration

这时候：MLE比Gradient Descent快得多

因为：一步出答案

(3). Linear Regression也有解析解

普通Linear Regression：y=X*θ + ϵ

假设：ϵ∼N(0,σ^2)

MLE推导后：

得到：θ=(X.T @ X)^−1 @X.T @y

这叫：Normal Equation

直接：theta = np.linalg.inv(X.T @ X) @ X.T @ y

也是：不需要Gradient Descent

(4). 那为什么还学Gradient Descent？

因为很多MLE问题：没有解析解

例如:Logistic Regression。

MLE目标：θ maxP(D∣θ)

推导后 得到：i∑​y_i *​logp_i​ + (1 − y_i​)*log(1 − p_i​)

这个式子：不能直接解出θ

于是只能：

猜一个θ

算gradient

更新θ

再算gradient
...

这就是：Gradient Descent

(5). 所以关系是
<pre>
MLE
 ↓
得到优化目标
 ↓
能直接解？
 ├─ YES → Closed Form
 └─ NO
      ↓
Gradient Descent
</pre>

你一直在：

你是在 用Gradient Descent求MLE

### 6. 为什么要学习各种 Distribution? 

因为不同数据类型，对应不同分布。每个模型背后, 都对应一个概率分布假设.

机器学习本质，先假设数据来自某种distribution，然后做MLE。

### Bernoulli Distribution：Logistic Regression = Bernoulli + MLE方法

例如：

是否点击广告
是否患病
是否通过考试

只有：0 或 1

使用：Bernoulli Distribution

MLE：P(y)=p^y * (1−p)^(1−y)

得到：p^ = #1/N

#### 为什么用log p probability 并且应用于所有的 Distribution？

抛硬币：9 heads  1 tail

Likelihood：P(9H,1T∣p=0.9)

根据 Bernoulli Distribution：L(p)=P(D∣p) = p^9(1−p)^1

*为什么不用这个直接求？*

因为当数据很多时,例如：1000次投硬币

Likelihood：p^900(1−p)^100 = 0.000000000000000... 电脑容易underflow（数值下溢）

所以取 Log 因为：log(ab)=loga+logb 乘法变加法。于是：logL(p)=log(p^9*(1−p)) = =9logp+log(1−p)

*为什么可以取 Log*

因为：

Log 是单调递增函数。例如：100 > 10  取log：log(100) > log(10)仍然成立。

所以：argmaxL(p)=argmaxlogL(p) 最大值位置不会变。

求MLE求导：导数​(9logp+log(1−p)) = 9/p + 1/1-p, 设为0：p = 0.9

*所以老师一直强调 Log 的原因有两个：*

数值稳定（避免极小数连乘）
乘法变加法，方便求导和优化

这也是为什么机器学习里几乎所有 MLE 推导最后都会出现 log-likelihood。

#### 为什么用负数 - log()？ 
原因1：机器学习喜欢“最小化”

MLE本来定义是：就是找argθ 的max P(D∣θ)

这是：最大化（maximize）

但是机器学习里的优化器：

GradientDescent
Adam
SGD
RMSProp

几乎全部设计成：最小化（minimize）

所以：大家习惯把：maxlogP()变成：min−logP(D∣θ)

加 log 是为了方便计算，
加负号是为了把“最大化概率”
变成“最小化损失”。

它其实就是：

MLE 换了个写法

本质仍然是在找：

最可能产生数据的参数

# Maximum A Priori (MAP) Estimation 看到新证据以后，更新自己的判断

普通 Machine Learning 在干嘛？ 例如 Linear Regression：

会找哪个参数最能 fit 数据  + 我只相信观测数据

这叫Maximum Likelihood Estimation（MLE）最大似然估计。

MAP 多了什么？ MAP 会说：除了数据，我还有一些“先验经验”。 例如：你训练一个模型。

MLE可能得到：w = 10000， 因为它能特别fit训练集。但你会觉得这参数也太离谱了吧，于是：MAP 会加入prior belief（先验），例如：我更相信参数应该小一点。于是不会只考虑fit data， 还会考虑：参数是不是合理。

一个非常重要的直觉，MAP：本质像“数据 + 常识”。MLE像：只看考试成绩，MAP像：考试成绩 + 平时表现。

<pre>
Distribution
      ↓
Likelihood P(D|θ)
      ↓
MLE
      ↓
得到模型参数

加入 Prior P(θ)
      ↓
MAP
      ↓
Regularization
</pre>

比如你在判断一枚硬币是不是公平。

### 贝叶斯公式分解

1. Prior：先验

在扔硬币之前，你心里有个默认想法：

大部分硬币应该差不多公平，正面概率大概接近 0.5。

这叫：P(θ)

也就是 prior。

θ 表示硬币正面概率。

2. Likelihood：数据支持程度

然后你开始扔硬币。

假设看到：

10次里面9次正面

你会问：

如果 θ=0.9，看到这个结果合理吗？

如果 θ=0.5，看到这个结果合理吗？

这叫：P(D∣θ)， 也就是 likelihood。

意思是：如果参数是 θ，现在这组数据出现的可能性多大？

3. P(D)数据D出现的总概率，也叫：Evidence（证据）或者：Marginal Likelihood

P(D) 的作用不是提供信息，而是把所有 posterior 归一化成真正的概率。

每个数据出现概率 × 先验概率全部加起来。

4. Posterior：看完数据后的新判断

贝叶斯说：

我不能只看原来的想法，也不能只看新数据。
我要把两者结合起来。

公式是：

看完数据后对 θ 的相信程度 = 数据对 θ 的支持 × 根据常识原本对 θ 的相信程度 ÷ 归一化常数

### MAP 是什么？

MAP 就是：

在 posterior 里面找最高点。

也就是：

看完数据以后，我最相信哪个 θ？

公式：θMAP​=argθmax​P(θ∣D)

因为 P(D) 对所有 θ 都一样，所以实际就是：θMAP​=argθmax​P(D∣θ)P(θ)

大白话： MAP = 找一个既能解释数据、又符合先验常识的参数

#### 第一步：回忆 MLE

假设：10 次投硬币, 8 Heads, 2 Tails

那么：nH​=8, nT​=2

MLE：θ_MLE​=nH​ / (nH​+nT​)​ = =8/10​=0.8, MLE认为：正面概率 = 80%

#### 第二步：MAP 多了 Prior

贝叶斯说：

我不完全相信这10次实验

因为：样本太少

所以给 θ 一个 Prior：θ∼Beta(α,β)

这里：α 代表：我先验认为有 α−1 次 Head

β 代表： 我先验认为有 β−1 次 Tail

一个大白话例子

假设：Beta(11,11) 什么意思？

相当于：在你真正开始实验前。我已经假装看到：

10 Heads
10 Tails

因为：

α−1=10
β−1=10

#### 第三步：真实数据来了

真实观察：8 Heads, 2 Tails

于是：

总Head：8+10=18

总Tail：2+10=12

MAP：θ= 18/30 = 0.6

注意：MLE 0.8  MAP 0.6 因为：先验把结果拉回去了

为什么公式长这样

<img width="337" height="88" alt="image" src="https://github.com/user-attachments/assets/73e87fe2-c489-432d-9dfa-66d2f971a5bb" />

(真实Head + 虚拟Head) / (真实总次数 + 虚拟总次数)

也即，(观察到的Head + 先验Head) / (观察总数 + 先验总数)

α 和 β 可以理解成什么？

最容易记：

α = 虚拟Head数量 + 1

β = 虚拟Tail数量 + 1

例如： Beta(1,1) 就是 α=β=1

例如 If you see 35 heads and 17 tails, and use a uniform prior of α=β=10, what is the MAP for the parameter? 

(35+10-1) / (35 + 10 - 1 + 17 + 10 -1 )

### 比较MLE和MAP

MLE：谁最能解释数据，我选谁。

MAP：谁既能解释数据，又符合我原本的相信，我选谁。
