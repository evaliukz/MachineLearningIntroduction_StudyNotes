# Maximum Likelihood Estimation（MLE）

### 1. MLE到底是什么？

MLE：Maximum Likelihood Estimation

中文：最大似然估计

核心思想：假设数据已经发生了，哪组参数最可能产生这些数据？

举个最简单的例子：你捡到一枚硬币。

连续扔10次：正 正 正 正 正 正 正 反 正 正

统计：9个正面 1个反面

现在问：这枚硬币正面概率 p 是多少？

### 2. MLE在干嘛

MLE会尝试：p=0.1 ? p=0.2 ? p=0.5 ? p=0.9 ?

看哪个最可能产生：9个正面 1个反面 如果：p=0.9 出现这种结果非常合理。

如果：p=0.1 几乎不可能。 所以：MLE：p^=0.9

### 3. 什么是 Likelihood？
  
Likelihood：P(D∣θ)  表示：给定参数 θ （就是w和b的组合），产生数据 D 的概率

例如：参数p=0.9 数据：9 heads  1 tail

Likelihood：P(9H,1T∣p=0.9)

MLE：就是找argθ 的max P(D∣θ)

### 4. 为什么机器学习都在做MLE

因为训练数据已经给你了。你不知道真正参数是多少。所以：你只能反推：什么参数最可能生成这些数据？这就是：Parameter Estimation

### 5. MLE和Gradient Descent的关系？

很多机器学习初学者都会把：MLE， Gradient Descent误以为是两种不同的方法。

其实：*MLE是目标，Gradient Descent是求解方法*

(1). MLE到底是什么？

MLE说：θ^=argθmaxP(D∣θ) 意思：找到最可能产生数据D的参数θ，这是一个优化目标，不是算法。

(2). 怎么找到这个参数？有两种情况。

情况1：有解析解（Closed Form Solution）

例如：估计高斯分布均值。

数据：1,2,3,4,5

MLE推导：直接得到：

μ^ = ∑x_i / n

直接算：m_u = np.mean(x)

结束。不需要：gradient descent, learning rate, iteration

这时候：MLE比Gradient Descent快得多

因为：一步出答案

(3). Linear Regression也有解析解

普通Linear Regression：y=X*θ + ϵ

假设：ϵ∼N(0,σ^2)

MLE推导后：

得到：θ=(X.T @ X)^−1 @X.T @y

这叫：Normal Equation

直接：theta = np.linalg.inv(X.T @ X) @ X.T @ y

也是：不需要Gradient Descent

(4). 那为什么还学Gradient Descent？

因为很多MLE问题：没有解析解

例如:Logistic Regression。

MLE目标：θ maxP(D∣θ)

推导后 得到：i∑​y_i *​logp_i​ + (1 − y_i​)*log(1 − p_i​)

这个式子：不能直接解出θ

于是只能：

猜一个θ

算gradient

更新θ

再算gradient
...

这就是：Gradient Descent

(5). 所以关系是
<pre>
MLE
 ↓
得到优化目标
 ↓
能直接解？
 ├─ YES → Closed Form
 └─ NO
      ↓
Gradient Descent
</pre>

### 6. 为什么要学习各种 Distribution? 

因为不同数据类型，对应不同分布。每个模型背后, 都对应一个概率分布假设.

机器学习本质，先假设数据来自某种distribution，然后做MLE。

| Model                      | Distribution |
| -------------------------- | ------------ |
| Linear Regression          | Gaussian     |
| Logistic Regression        | Bernoulli    |
| Poisson Regression         | Poisson      |
| Multinomial Classification | Multinomial  |


### Bernoulli Distribution：Logistic Regression = Bernoulli + MLE方法

例如：

是否点击广告
是否患病
是否通过考试

只有：0 或 1

使用：Bernoulli Distribution

MLE：P(y)=p^y * (1−p)^(1−y)

得到：p^ = #1/N

为什么用log p probability 尤其是toss coin Bernoulli Distribution

抛硬币：9 heads  1 tail

Likelihood：P(9H,1T∣p=0.9)

根据 Bernoulli Distribution：L(p)=P(D∣p) = p^9(1−p)^1

*为什么不用这个直接求？*

因为当数据很多时,例如：1000次投硬币

Likelihood：p^900(1−p)^100 = 0.000000000000000... 电脑容易underflow（数值下溢）

所以取 Log 因为：log(ab)=loga+logb 乘法变加法。于是：logL(p)=log(p^9*(1−p)) = =9logp+log(1−p)

*为什么可以取 Log*

因为：

Log 是单调递增函数。例如：100 > 10  取log：log(100) > log(10)仍然成立。

所以：argmaxL(p)=argmaxlogL(p) 最大值位置不会变。

求MLE求导：导数​(9logp+log(1−p)) = 9/p + 1/1-p, 设为0：p = 0.9

# Maximum A Priori (MAP) Estimation

普通 Machine Learning 在干嘛？ 例如 Linear Regression：

会找哪个参数最能 fit 数据  + 我只相信观测数据

这叫Maximum Likelihood Estimation（MLE）最大似然估计。

MAP 多了什么？ MAP 会说：除了数据，我还有一些“先验经验”。 例如：你训练一个模型。

MLE可能得到：w = 10000， 因为它能特别fit训练集。但你会觉得这参数也太离谱了吧，于是：MAP 会加入prior belief（先验），例如：我更相信参数应该小一点。于是不会只考虑fit data， 还会考虑：参数是不是合理。

一个非常重要的直觉，MAP：本质像“数据 + 常识”。MLE像：只看考试成绩，MAP像：考试成绩 + 平时表现。

<pre>
Distribution
      ↓
Likelihood P(D|θ)
      ↓
MLE
      ↓
得到模型参数

加入 Prior P(θ)
      ↓
MAP
      ↓
Regularization
</pre>
