# 什么是 Linear Squares Regression?

Linear Least Squares Regression，中文通常叫：线性最小二乘回归
它本质是在做一件事：找一条线，让所有点到这条线的“误差平方和”最小。
应用场景：
•	根据房屋面积预测价格
•	根据广告费用预测销售额
•	根据温度预测冰淇淋销量
 
### 最小二乘法的核心思想
我们希望找到一组 w 和 b，使得预测值和真实值之间的差距（即误差）平方和最小。
y=wx+b

这里：
x：输入，比如学习小时数
y：真实分数
y^：模型预测分数
w：斜率，表示 x 每增加 1，y 大概增加多少
b：截距，表示 x=0 时的基础值

### 什么叫 Least Squares

每个点都会有误差：error=y−y^
如果直接把 error 加起来，正负会抵消，所以我们平方：
Least Squares 的意思就是：让所有 error² 加起来最小

为什么用平方？主要有三个原因：
第一，平方后误差不会正负抵消。
第二，大错误会被惩罚得更重。比如 error=10，平方是100。
第三，平方函数很好求导，方便用数学优化。

# Decision Boundary Geometry（决策边界几何）

### 什么是 Decision Boundary

Decision Boundary（决策边界）就是：

“模型用来把不同类别分开的边界线/边界面。”

例如：

- 边界左边 → 猫
- 边界右边 → 狗

本质上：

模型是在问：“这个点应该分到哪边？”

## 2. Linear Model 的公式

线性模型通常长这样：

f(x)=w^Tx+b


其中：

- \(x\)：输入特征
- \(w\)：权重（weights）
- \(b\)：bias/intercept

分类时：

- \(f(x)>0\) → class 1
- \(f(x)<0\) → class 0

真正的分界线：

\[
w^Tx+b=0
\]

Decision Boundary

### Geometry（几何意义）

二维时


w_1x_1+w_2x_2+b=0

是一条：直线

三维时

是一张：平面

更高维是：Hyperplane（超平面）

### 最核心理解：w 是边界的法向量（normal vector）w 指向 boundary 最陡上升方向。二维的-w1/w2是decison boundary的斜率。

权重向量：

w=(w_1,w_2,...)

决定：边界朝向哪里

并且：决策边界永远垂直于 w

### 给定w和b如何计算decision boundry？

w = (1,-3)

b = 1

#### Step 1: 先写成公式

For a linear classification model: f(x)=w^Tx+b

The decision boundary is where: w^Tx+b=0

#### Step 2: Expand the equation 

w^Tx = 1 * x_1 + (-3)* x_2

x_1 - 3x_2 + 1 = 0

#### Step 3: Convert into line form 改写成直线形式

Solve for \(x_2\):

x_1 + 1 = 3x_2

x_2 = 1/3 * x_1 + 1/3 

也可以用 x_1 = 3* x_2 - 1

This is the decision boundary line.

#### Step 4: Understand the geometry

## Slope

m=1/3

The line slopes upward slowly.


####  Intercept

1/3

So the line crosses the y-axis at:

(0,1/3)

####  Step 5: Determine which side is positive

The classifier predicts based on:

f(x)=x_1-3x_2+1

- If:f(x)>0 → positive class

- If:f(x)<0 → negative class

####  Step 6: Test a point

Try the origin:
(0,0)

Substitute into the function:

0-0+1=1


Since:1>0
the origin is on the positive side of the boundary.

# Convex Surrogate Loss Functions
## 1. 为什么需要 Convex Surrogate Loss

分类问题真正想优化的是：Classification Error（分类错误率）

也就是：

- 分类正确 → loss = 0
- 分类错误 → loss = 1

对应：Zero-One Loss
\ell^{0,1}=1[y\hat y<0]

其中：
- \(y\in\{-1,+1\}\)：真实标签
- \(\hat y\)：模型输出

如果：y\hat y>0 说明预测正确。

如果：y\hat y<0 说明预测错误。

Zero-One Loss 的问题是：不连续、不可导

像台阶函数一样：
对 → 0
错 → 1
因此 无法直接用 Gradient Descent 优化

### Convex Surrogate 的核心思想

于是机器学习中，不直接优化 0-1 loss，而是用“可优化”的 convex loss 去近似它。

这些替代函数：
平滑
可导
凸（convex）
能用梯度下降优化

所以叫 Convex Surrogate Loss Functions
Convex（凸）是什么意思

Convex function：

像“碗”一样。

特点：

没有复杂局部最优
Gradient Descent 更稳定
更容易找到全局最优

## 手写zero one loss：
先理解 Zero-One Loss

分类正确loss = 0

分类错误loss = 1

为什么：y*y^<0表示分类错误？

因为：如果真实label y=1 预测：y^>0 则：y*y^>0 正确。

如果真实label：y=1 预测：y*y^<0表示分类错误

本质：

同号 = 正确
异号 = 错误

np.sign 是什么？

x < 0 → np.sign(x) = -1

x = 0 → 0

x > 0 → 1

<pre>
 import numpy as np

def zeroOneLoss(predictFunc, features, labels, params):

    """
    predictFunc:
        prediction function

    features:
        input X

    labels:
        true labels (-1 or +1)

    params:
        model parameters
    """

    # Step 1:
    # 用模型预测
    yhat = predictFunc(features, params)

    # Step 2:
    # 把预测值变成:
    # -1 / 0 / +1
    pred_sign = np.sign(yhat)

    # Step 3:
    # 判断哪些预测错误
    #
    # pred_sign != labels
    #
    # True  -> prediction wrong
    # False -> prediction correct
    errors = pred_sign != labels

    # Step 4:
    # True=1 False=0
    #
    # 所以mean就是:
    # average classification error
    loss = np.mean(errors)

    return loss
</pre>

### Cosine Learning Rate Decay（余弦学习率衰减）

为什么不用直线下降？ learning rate下降得太生硬
但实际训练：

希望前期下降慢一点 方便：快速探索。

中期下降快一点 方便：快速收敛。

后期下降再慢一点 方便：稳定逼近最优点。

于是：cosine curve 很适合。

cosine curve 长什么样

cos(x)从：0→π/2时：起点 cos(0)=1 终点cos(π/2)=0

图像：开始平缓 中间下降较快 最后再次平缓 这非常适合learning rate schedule。

公式拆解

你写的：η=η0 * cos(t/T * π/2) )+ηf
	​
t/T 是什么? 表示：当前训练进度。 ∈[0,1]

例如：

开始t=0
中间t=T/2
结束t=T

为什么乘：π/2

因为：想让起点cos(0)=1,终点cos(π/2)=0

于是：learning rate：从 eta0 平滑降到 etaf。

# 最小二乘回归（Linear Least Squares Regression）中L1 和 L2 regularization正则化

问题：在LSR里当特征太多 or 训练数据太少，会发生什么？
•	模型容易过拟合（在训练集上表现很好，在新数据上表现差）
•	权重 w会变得不稳定（尤其在特征共线性时）
解决方案：加上 正则项（Regularization Term）
什么是正则化？
正则化 = 惩罚复杂模型，鼓励简单权重（小且稀疏）
我们在原始损失函数后面加一个“惩罚项”来约束权重 w，防止它们变得太大或太复杂。
L2 Regularization（也叫 Ridge Regression）
 
解释：
•	惩罚所有权重过大的情况
•	越大的 w，惩罚越重
•	结果：权重被压小，但通常不会完全变成 0
•	有效地 降低模型复杂度，改善泛化能力
•	后边的部分，lambda部分是一个circle constraint 因为wi^2加总 < 一个constraint

效果：
•	收缩（shrinkage）模型，但不会让特征“消失”
•	更适用于所有特征都有用，但不希望它们太激进

Lambda或者wi 越大，后面的权重越大，是一个固定的数，前面的x和y的影响越小。曲线越扁，因为后边权重是个constant，当权重lambda无限大时候，整个曲线最后会变成一个横线。
 

L1 Regularization（也叫 Lasso Regression）
 
解释：
•	惩罚权重的“绝对值”
•	结果：部分权重会被压成 0
•	即：自动选择特征（特征选择能力）
•	后边的部分，lambda部分是一个diamond constraint
•	 
效果：
•	更适用于数据中存在很多冗余特征时
•	能“丢弃”一些无关特征 → 稀疏模型

# 特性	L1 正则化（Lasso）	L2 正则化（Ridge）
惩罚形式	 sum	w_i
结果权重	稀疏（可能为 0）	收缩（趋向于小）
特征选择能力	✅ 自动剔除不重要的特征	❌ 不会将权重设为 0
稳定性	对噪声敏感（不稳定）	更稳定，抗多重共线性
解法	不可导（需迭代优化）	可直接求闭式解（解析解）
适用场景	特征多，但只有少数有用	所有特征都有用，但需抑制过拟合
•  特征非常多，可能很多无用 → L1（Lasso）
•  想让模型更稳定，避免权重过大 → L2（Ridge）
•  想要两者兼顾 → 使用 Elastic Net，结合了 L1 + L2
通常我们应该 exclude（排除）bias term（偏置项）不参与正则化
问题	答案
需要正则化 bias term 吗？	❌ 通常不需要
为什么？	避免不必要地抑制输出偏移；提高模型表现
在代码里需要特别排除吗？	如果手动实现，需要排除；库（如 sklearn）会自动处理

1. 偏置项的作用不同于权重项
•	权重项 w 控制模型对输入特征的响应；
•	偏置项 b 控制模型的输出整体偏移（intercept）；
•	对偏置项加正则化可能会导致模型在整体输出上出现不必要的偏移。
2. 防止欠拟合
如果你正则化偏置项（也就是惩罚它），在某些情况下会导致模型整体能力下降：
•	输出容易被拉向 0；
•	模型欠拟合的可能性增加，特别是在数据不以原点为中心时。
